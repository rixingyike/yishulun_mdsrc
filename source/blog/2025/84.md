# 我准备写一个梯子软件，主要自用

今天在微信群里，墨友们讨论了梯子的问题，大家都明显感受到最近在使用 AI IDE 的时候，需要点 retry 或 resume 的时刻变多了。出问题时看到的最多的一个提示是，区域不匹配！

为什么区域不匹配？因为谷歌的服务器服务谁不服务谁是看区域的，台湾服务，香港服务，日本服务，但就是中国大陆不服务。当看到区域不匹配的时候，完了，就是你的梯子把你出卖了，谷歌识破了你的伪装。

伪装与反伪装是怎么回事呢？当前阶段主要是指纹识别。我们作为网民，与服务器进行数据交互，在服务器看来，我们的客户端是有指纹的。什么 IP，什么系统，什么浏览器，什么版本，什么协议，这些信息都是指纹信息的一部分。这些信息汇成一体生成 JA3 发送给服务器，服务器一看，咦，怎么这么多重复的 JA3？这里面肯定有问题！

为什么会有重复或高度相似的 JA3？因为许多人购买的是同一个梯子，但他们虽然在不同地方上网，但都是通过一种软件，在某一个服务器上跳转，向服务器发出请求的，加上软件开发者对指纹处理的不作为或忽视，服务器能不发现呢？

现代服务器识别用户端是不是重复访问，或恶意抓取信息，用的技术主要也是指纹（FingerPrinting）识别。像 Cloudflare 它防机器人，用的主要也是指纹识别这项技术。刷软件流量做得好的团队，都懂得伪造指纹，把发向服务器的请求，做成 Chrome 或 Edge 请求，把请求本身自带的指纹伪造得和真人一样，再加上 IP 也是随机的，那基本上服务器就识别不出来了，请求虽然来自同一台客户端电脑，但在服务器看来，就是来自不同地方的不同电脑。

我想写一个梯子软件，默认直接开 Tunnel 模式，本地创建虚拟网卡，通过 Quic 协议与服务器连接。今天刚写了一个小 demo，客户端 client 放在本地，server 放在美国服务器上，已经跑通了，如图所示。

但离一个真正的软件还有许多事要做。最重要的，就是前面我说的指纹伪造。有的梯子堂而皇之在请求头中标上自己的软件，这种就是傻，服务器见多了，一下就能统计出规律来。还有的梯子，本来挺隐蔽，但因为用的人多了，就不隐蔽了。自己写一个梯子的好处就是，它是独特的，外面没有，服务器很难怀疑到它。还有就是，即使要给不同的朋友使用，我也让每个 client 端伪造出不同的指纹出来。不要有自己的软件头，伪造成普通的浏览器流量，像真人一样，带着不同的指令信息，向服务器发出请求。这样，即使使用这款软件的朋友分布在不同地方，服务器也很难发现有问题。

至于 Global、Direct、Rules 三模式切换，及 UI、及用户权限系统的开发，这些都属于常规的末节了，都是相对比较简单的。
