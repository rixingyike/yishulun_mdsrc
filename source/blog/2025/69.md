# 墨问终端脚本再添新功能，自动分发至博客园、自搭博客

博客园支持 MetaWeblog API，只需要在终端环境里设置：
```
# 博客园配置（从环境变量获取）
CNBLOG_BLOG="${CNBLOG_BLOG_NAME:-}"
ACCOUNT="${CNBLOG_BLOG_ACCOUNT:-}"
PASSWD="${CNBLOG_BLOG_PASSWD:-}"
```
就可以自动同步至博客园社区。该社区是老牌技术社区，站长杜勇还得过盖茨赠予的 Windows 3.1 系统盘。该社区的优势是 PR 值高，估计在 7 甚至更高，同样的内容，同一个作者所写，博客园的链接在搜索引擎排名中会靠前。

除了支持博客园发布，还支持自建博客发布。我的博客是用 rustpress 搭建的。我为什么知道 rustpress 博客支持同步？因为这个 Rust 程序就是我编写的。不需要任何额外设置，提交即可。

rustpress 博客是静态博客，使用 md 作为博客源码，将文章提交到 github 仓库，通过 github Action 博客即会自动编译。关于 github workflow deploy.yml 在开源仓库里有示例，配置很简单。rustpress 开源地址：

[https://crates.io/crates/rustpress](https://crates.io/crates/rustpress)

它已经发布在 crates 网站上，使用它也只需要一个 rust install 指令。

使用自动同步就不要太介意自己撰写的内容扩散，另外在撰写时，也要更加注意措辞，因为扩散后修改会比较麻烦。脚本有自动扩散功能，还没有自动修改同步功能。下面是昨天撰写的菜豆腐米饭笔记，页面呈现还可以，图片也是自动上传的。

[https://www.cnblogs.com/sban/p/19343981](https://www.cnblogs.com/sban/p/19343981)

每个平台用的都是自己的图床，我不会把图片上传在墨问的 cdn 上，然后用线上链接同步至其他平台，这样做虽然省事，但极易遭到平台嫌弃。许多平台都不喜欢其他平台承载自己应该承载的图片流量，在发现了是其他平台的图片后，有时候还会显示替代提示符。常在互联网圈，图片更新我不会犯错误的，在谁家发布，就用谁家的图床。

bye，就聊这么多，祝使用脚本撰写愉快。
